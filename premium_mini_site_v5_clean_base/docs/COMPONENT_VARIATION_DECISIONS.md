# Component Variation Decisions (V5)

Documentation of the 21st.dev-style variation workflow for V5 master components.

| Component | Variations Considered | Selected Direction | Conversion Rationale |
| :--- | :--- | :--- | :--- |
| **IntentHero** | A: SaaS Split, B: Typographic Center, C: Editorial Asymmetric | **C: Editorial Asymmetric** | Distinguishes the brand from cheap templates; high visual authority. |
| **DecisionDock** | A: Segmented Control, B: Icon Grid, C: Floating Tactile Pill | **C: Floating Tactile Pill** | Thumb-optimized; reduces friction for intent selection on Android. |
| **OfferBoard** | A: Feature Grid, B: Pricing Stack, C: Decision Cards | **C: Decision Cards** | Helps user self-segment into specific service tiers. |
| **VisualNarrative** | A: Standard Grid, B: Masonry Collage, C: Horizontal Rail | **B: Masonry Collage** | Superior aesthetic for Instagram-based brands; handles varied aspect ratios. |
| **ObjectionResolver**| A: Accordion List, B: Card Wall, C: Interactive Briefing | **C: Interactive Briefing** | Proactively resolves fears near decision points. |
